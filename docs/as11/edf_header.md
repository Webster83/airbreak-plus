# AirSense 11 EDF Header Reference

This document describes the fixed EDF header fields used by AS11 `BRP`,
`PLD`, `SA2`, `STR`, `EVE`, and `CSL` files.
Signal definitions are listed in \
[AirSense 11 EDF Signal Reference](edf_signals.md). \
EDF+ annotation payloads are described in \
[AirSense 11 EDF Annotation Reference](edf_annotations.md).

## Fixed header

AS11 uses the normal 256-byte EDF fixed header. Text fields are ASCII,
left-aligned, space-padded, and not NUL-terminated.

| Offset | Size | Field | AS11 content |
|--------|------|-------|--------------|
| `0x000` | 8 | version | `0` |
| `0x008` | 80 | patient ID | see [Patient ID](#patient-id) |
| `0x058` | 80 | recording ID | see [Recording ID](#recording-id) |
| `0x0a8` | 8 | start date | `dd.mm.yy` |
| `0x0b0` | 8 | start time | `hh.mm.ss` |
| `0x0b8` | 8 | header byte count | decimal, usually `256 + 256 * signals` |
| `0x0c0` | 44 | reserved | `EDF` for sampled/stat files, `EDF+D` for `EVE`/`CSL` |
| `0x0ec` | 8 | data record count | decimal |
| `0x0f4` | 8 | data record duration | decimal seconds with two fractional digits |
| `0x0fc` | 4 | signal count | decimal |

Common AS11 values:

| File | Reserved | Record duration | Notes |
|------|----------|-----------------|-------|
| `BRP.edf` | `EDF` | `60.00` | sampled waveform records |
| `PLD.edf` | `EDF` | `60.00` | sampled low-rate records |
| `SA2.edf` | `EDF` | `60.00` | sampled oximetry records |
| `STR.edf` | `EDF` | `86400.00` | session summary records |
| `EVE.edf` | `EDF+D` | `0.00` | EDF+ annotation records |
| `CSL.edf` | `EDF+D` | `0.00` | EDF+ annotation records |

Example sampled-file fixed header fields:

```text
version      "0       "
patient_id   "X X X X 4AED D4BA                                                               "
recording_id "Startdate 10-MAY-2026 X X X SRN=23252392262 MID=46 VID=7                        "
start_date   "10.05.26"
start_time   "04.12.26"
header_bytes "1024    "
reserved     "EDF                                         "
records      "82      "
duration     "60.00   "
signals      "3   "
```

## Patient ID

AS11 patient ID has the form:

```text
X X X X %04X %04X
```

The `X X X X` prefix is literal. The two four-hex-digit fields are CRC16
values generated by the EDF writer:

| Word | Source bytes | Notes |
|------|--------------|-------|
| first `%04X` | fixed header bytes `0x19..0xff` | starts immediately after the second word |
| second `%04X` | signal header block bytes `0x100..header_bytes-1` | all EDF per-signal metadata blocks |

Both use CRC16-CCITT-FALSE

## Recording ID

AS11 recording ID has the form:

```text
Startdate DD-MMM-YYYY X X X SRN=<serial> MID=<mid> VID=<vid>
```

Fields:

| Field | Meaning |
|-------|---------|
| `DD-MMM-YYYY` | file start date using uppercase English month names |
| `SRN` | device serial number |
| `MID` | platform/model identifier; known AS11 files use `46` |
| `VID` | variant identifier from the active firmware/device identity |

## Signal header blocks

After the 256-byte fixed header, EDF stores one block per signal metadata
field. Each block contains all signals before the next metadata field starts:

| Order | Width per signal | Field |
|-------|------------------|-------|
| 1 | 16 | label |
| 2 | 80 | transducer type |
| 3 | 8 | physical dimension/unit |
| 4 | 8 | physical minimum |
| 5 | 8 | physical maximum |
| 6 | 8 | digital minimum |
| 7 | 8 | digital maximum |
| 8 | 80 | prefiltering |
| 9 | 8 | samples per data record |
| 10 | 32 | reserved |

Total header size is therefore:

```text
256 + 256 * signal_count
```

`Crc16` signal metadata:

| Field | Value |
|-------|-------|
| label | `Crc16` |
| unit | blank |
| physical minimum | `-32768.0` |
| physical maximum | `32767.00` |
| digital minimum | `-32768` |
| digital maximum | `32767` |
| samples per record | `1` |

For `EVE.edf` and `CSL.edf`, signal 0 is `EDF Annotations` and signal 1 is
`Crc16`; see [AirSense 11 EDF Annotation Reference](edf_annotations.md).
